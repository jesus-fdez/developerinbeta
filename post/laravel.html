Laravel es un framework de PHP robusto y elegante para desarrollar aplicaciones web modernas. En esta guía aprenderás a instalar Laravel 12 (versión actual) utilizando composer, el gestor de dependencias para PHP.

Requisitos Previos

Antes de comenzar, asegúrate de que tu sistema cumpla con los siguientes requisitos:

  • PHP instalado en tu máquina. Te dejo la página de instalación y configuración de PHP aquí
  • Acceso a una terminal o línea de comandos.
  • Permisos de administrador para instalar software.

Instalación de Composer

Composer es una herramienta esencial para gestionar las dependencias en proyectos PHP. Aquí podrás encontrar la documentación oficial de Composer, donde puedes seguir el tutorial para su instalación.

Es sencillo, básicamente es instalar un .exe si el sistema es Windows, o lanzar un par de comandos (recomiendo instalarlo de forma global) si tienes un sistema basado en Unix.

Instalación de Laravel

Ya tenemos composer, para asegurarnos podrás ejecutar en un terminal composer -v y debe salir algo como la siguiente imagen:

La documentación oficial de Laravel indica que se instale la herramienta "Laravel Installer", pero personalmente uso Composer tanto para la creación del proyecto como para la gestión de dependencias. No uso "Laravel Installer" para crear el proyecto Laravel, pero para gusto los colores.

Con Composer instalado, ahora puedes proceder a instalar Laravel con estos pasos:

1. Crear un nuevo proyecto Laravel

Ejecuta el siguiente comando en tu terminal para crear un nuevo proyecto de Laravel:

composer create-project laravel/laravel nombre-del-proyecto

Reemplaza nombre-del-proyecto con el nombre que desees para tu aplicación. Este comando descargará e instalará Laravel y todas sus dependencias en un nuevo directorio con el nombre especificado.

2. Generar la clave de la aplicación

Accede al directorio de tu proyecto:

cd nombre-del-proyecto

Fíjate si al finalizar la creación del proyecto, ha lanzado el siguiente comando:

@php artisan key:generate --ansi

Seguramente lo encuentres, es decir, ha generado la clave de la aplicación. Por si acaso en futuras versiones no lo hace al generar el proyecto, te dejo por aquí el comando para que la puedas generar tú:

php artisan key:generate

Esto generará un hash como valor para APP_KEY en tu fichero .env, para cifrar los valores en tu proyecto Laravel. Si vas a usar base de datos, deberás configurar los valores necesarios en este fichero de entorno para la conexión a tu base de datos. Ejemplo para Mysql:

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=database
DB_USERNAME=root
DB_PASSWORD=password
3. Iniciar el Servidor de Desarrollo

Para iniciar el servidor de desarrollo integrado de Laravel, ejecuta:

php artisan serve

Esto iniciará un servidor en http://localhost:8000. Puedes acceder a esta URL en tu navegador para ver la aplicación en funcionamiento.

Has aprendido a instalar Composer y Laravel en tu sistema. Como verás, en cuestión de minutos tienes una aplicación lanzada en un servidor en local, preparada para desarrollar. Bootstrap auth scaffolding

En mi caso elijo bootstrap porque junto con Tailwind Css, me parece lo mejor para proyectos pequeños. Supongo que según las necesidades de tu proyecto podrás elegir otro framework de front con el que generar el código.

Ejecuta el siguiente comando para instalar el paquete Laravel UI, dentro del directorio del proyecto:

composer require laravel/ui

A continuación usa el siguiente comando del paquete Laravel UI, para generar el código de autenticación usando Bootstrap 5: 

php artisan ui bootstrap --auth

Sí en consola aparece esta pregunta:

The [Controller.php] file already exists. Do you want to replace it? (yes/no) [yes]

Pulsa enter, yes por defecto.

Ya tenemos el código. Las últimas versiones de Laravel usan NPM para instalar los paquetes/dependencias frontend, necesitamos instalarlo (aunque si lo prefieres se puede trabajar de manera "tradicional" sin npm). Dejo aquí la documentación oficial donde se detallan los pasos para su instalación.

Ahora sí, con el entorno ya preparado, ejecuta el siguiente comando para instalar las dependencias y recargar/reemplazar los módulos "en caliente": 

npm install
npm run dev
